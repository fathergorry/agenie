**Быстрый старт**
1. Установка: 
- установить node.js 20+ (рекомендуется через стандартный установщик)
- скачать agenie - архивом, через npm или git 
- открыть консоль (cmd, powershell, bash или иную), перейти в каталог agenie (на уровень, где package.json). В Windows работает Shift+ПКМ -> Открыть в терминале.
- в каталоге agenie выполнить "npm i -g". Это сделает скрипт ag глобальным, т.е. доступным в любом каталоге.
- Перейти в каталог со СВОИМ проектом и выполнить ag install, это создаст директорию ag-workspace и файлы для работы с проектом 
2. Купить доступ к нейросети или агрегатору нейросетей, или получить иным способом. Сеть должна работать по протоколу openAI, и вам должны будут выдать 
- ключ - строку вида "sk-or-vv-...."
- адрес точки доступа
3. Открыть в текстовом редакторе ag-workspace/_config.js и вставить туда оба этих параметра вместо имеющихся, например:
    opts.apiKey = "sk-or-vv-123456789012345" 
    opts.url = "https://api.vsegpt.ru/v1/chat/completions"
Сохранить.
4. Открыть ag-workspace/_tasks.log и с новой строки написать:
@PROMPT listing==none
Создай в ./sandbox/ два файла с короткими стишками.
5. В консоли, в каталоге вашего проекта (рядом с /ag-workspace/ но не внутри нее) выполнить команду ag task. Программа создаст нужные файлы и комментарий к вашему промпту.


**Описание**

Agenie - ИИ-агент на node.js, умеющий
- Создавать проекты на любом доступном LLM языке
- Внедряться в имеющийся проект, добавлять фичи и делать рефакторинг, всё под контролем разработчика (прощай, legacy-геморрой)
- Прозрачно работать с консолью

Рреализующий такую идеологию: 
1. Исходит не из плана и истории, а из текущего состояния проекта и задач - таким образом, требуя минимального сопровождения документацией. Буддийский подход: важно только то что есть сейчас. Это очень быстро, и будет надежно, если освоить особенности тестирования.
2. Минимум зависимости от поставщика услуги. Проект можно дробить на блоки и каждому блоку назначать свою LLM, а значит, ни одна из них не сможет его "украсть".
3. Минимум зависимостей в коде, максимальная простота. Точность и эффективность достигается не бюрократией, а минимализмом. (Здесь опытные разработчики, скорее всего, будут протестовать, но надо понимать, что привычный им избыток регуляций для быстрорастущего проекта - это плохо.)
4. Работает не вместо, а вместе с разработчиком, повышая ваш уровень и компетенции. 
5. Вместо подхода мелких инкрементов и вдумчивого тестирования - толерантность к ошибкам и гигантские изменения за малое время.

Рекомендуемый стартовый уровень для программиста - Middle Если ваш уровень меньше, рекомендуется его быстро поднять при поддержке agenie и техник раннего внимания, например от Нейробуддизма.

**Использование**

У вас будет 5 методов, доступных как функции или скрипты: task, diff, rollback, console и install

task - основное. Отправляет задание в LLM, получает результат, пишет в файлы. 
Задание вы описываете в основном журнале проекта _tasks.log, туде же дописывается и результат
Он организован как чат с нейросетью. Можно в текстовом редакторе создавать новые записи на основе старых, или с нуля. Старые записи используются только как источник для создания новых, или их можно удалять._

diff - просмотр текущих изменений от LLM

rollback - откат резервных копий. Здесь и в diff используются данные из @RESPONSE files в tasks.
Проект не использует git, так что для своего комфорта можете сделать commit перед использованием agenie

console - консольный нейросетевой агент, придумывает команды согласно задаче (вы всегда их подтверждаете), читает результат исполнения. Может как дополнять task так и работать самостоятельно

_config.js - здесь описываются:
- блоки проекта (чтобы не отправлять в LLM его целиком - это затратно и потенциально небезопасно)
- вызовы API используемых нейросетей
и т.д.

Скрипты устанавливаются вместе с самим модулем: npx ag-task, npx ag-diff, npx ag-rollback, npx ag-install
