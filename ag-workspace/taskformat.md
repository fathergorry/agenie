Файл описания задач для agenie представляет собой подобие чата где чередуются постановка задачи и отчет

@PROMPT param1==value1 param_multi==val1,,val2
Содержание задачи

@RESPONSE param==value
Комментарий к ответу

@PROMPT listing==main
Задача следующего блока... 
@RESPONSE files=./tests/main.test.js
Задача решена так

Для работы нам всегда нужен только последний блок, начинающийся с \n@PROMPT
После парсинга все параметры попадают в один JSON.
Строку от @ до \n разбиваем сначала по первому пробелу, а затем по шаблону AZaz_ + == + * (включая промежуточные пробелы)
Первая часть становится параметром, вторая - либо строковым значением, либо, если параметр входит в список массивных, то массивом из одного или более значений, значения разделены двойной запятой. 
Всё что идет от конца @-строки до \n, попадает в параметры. Это касается всех строк, начинающихся с @PROMPT @RESPONSE и др.
Дублирующие параметры перезаписываются. 
Таким образом, второй блок после парсинга будет выглядеть так: 
{
	PROMPT: "Задача следующего блока... ",
	listing: "main", 
	files: ["./config.js"], 
	tests: ["./tests/main.test.js"],
	RESPONSE: "Задача решена так"
}
